java集合与线程

面向对象

继承

提高代码复用性，继承是多态的前提。

封装

隐藏对象的属性和实现细节，仅对外提供公共访问方式。

多态

父类或接口定义的引用变量可以指向子类或实现类的实例对象。

集合

储存对象，长度可变，储存对象的类型可不同。

List

有序的，元素可重复，有索引。

ArrayList

底层是数组结构，增删慢，查询快，不同步。

LinkedList

底层是链表结构，增删快，查询慢，不同步。

Vector

底层是数组结构，线程同步，被ArrayList取代。

Set

无序，元素不可重复，无索引。

HashSet

底层是hash表，线程不同步，无序，高效。

元素唯一性：hashcode&&equals

LinkedHashSet

有序，是HashSet的子类。

TreeSet

底层是二叉树，可对元素进行排序，默认自然顺序。

排序

1，对象具备比较性

对象实现Comparable接口，实现comparaTo方法。

2，对象不具备比较性，或不是想要的比较方式（优先）

将Comparator比较器作为参数传入。

Map

Map取出方式：

1，取出key，放到一个Set中
Set<String> keySet=map.keySet();

2，取出键值对映射，放到一个Set中
Set<Map.Entry<String,String>> entrySet=map.entrySet();

HashMap

底层是hash表，允许使用null键null值，不同步，效率高。

TreeMap

底层是二叉树，不同步，可排序

HashTable

底层是hash表，不可存入null键null值，同步。

多线程

进程于线程

1，进程是静态的，就是开的一个程序。线程是动态的，是真正的执行单元和执行过程。
2，线程是划分得比进程更小的执行单位。
3，进程有一段专属的内存区域，而线程却共享内存单元，通过共享的内存单元来实现数据交换，实现通信与必要的同步操作。

创建线程

1，实现
new Thread(new Runnable() {
        @Override
        public void run() {

        }
    }).start();
2，继承
new Thread(){
        @Override
        public void run() {
            super.run();
        }
    }.start();

线程状态

新建

start()

临时状态

具备cpu执行资格，无执行权

运行状态

具备cpu执行资格，可执行

冻结状态

通过wait或sleep使线程不具备执行资格，需notify唤醒，并处于临时状态。

消亡状态

run方法结束，或销毁线程，使线程死亡。

线程安全

多线程，操作同一对象时，需要加锁。synchronized或lock

同步代码块
synchronized(对象){}

同步函数
private synchronized void xx(){}

wait和sleep区别

wait

可指定等待时间，如果不指定，将由notify或notifyAll唤醒，线程会释放执行权，且释放锁。

sleep

必须制定睡眠时间，时间到了自动处于阻塞状态。即使睡眠了，仍然持有锁，不会释放执行权。
