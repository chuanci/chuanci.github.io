smali

发表于 2016-12-11

Smali，Baksmali分别是指安卓系统里的Java虚拟机（Dalvik）所使用的一种.dex格式文件的汇编器，反汇编器。

数据类型

原始数据类型
    V                      void
    Z                      boolean
    B                      byte
    S                      short
    C                      char
    I                      int
    J                      long
    F                      float
    D                      double

引用数据类型
    L                      对象
    [                      数组

寄存器与变量

Android是基于寄存器存储的，一个寄存器占32位，java中long和double占8个字节，每一字节占8位，所以要用2个寄存器存储。

寄存器命名

v表示本地寄存器
p表示参数寄存器

如果一个方法有两个本地变量，有三个参数:

v0 第一个本地寄存器 
v1 第二个本地寄存器 
v2 p0 (this) 
v3 p1 第一个参数 
v4 p2 第二个参数 
v5 p3 第三个参数

当然，如果是静态方法的话就只有5个寄存器了，不需要存this了

.registers 使用这个指令指定方法中寄存器的总数
.locals 使用这个指定表明方法中非参寄存器的总数，放在方法的第一行。

语法

.field 　定义变量

.method　　方法

.parameter　　方法参数

.prologue　　方法开始

.line 12　　此方法位于第12行

invoke-super　　调用父函数

const/high16  v0, 0x7fo3　　把0x7fo3赋值给v0

invoke-direct　　调用函数

return-void　　函数返回void

.end method　　函数结束

new-instance　　创建实例

iput-object　　对象赋值

iget-object　　调用对象

invoke-static　　调用静态函数

跳转语句

"if-eq vA, vB, :cond_0" 如果vA==vB则跳转到:cond_0

"if-ne vA, vB, :cond_0" 如果vA!=vB则跳转到:cond_0

"if-lt vA, vB, :cond_0" 如果vA<vB则跳转到:cond_0

"if-gt vA, vB, :cond_0" 如果vA>vB则跳转到:cond_0

"if-le vA, vB, :cond_0" 如果vA<=vB则跳转到:cond_0

"if-ge vA, vB, :cond_0" 如果vA>=vB则跳转到:cond_0

"if-eqz vA, :cond_0" 如果vA==0则跳转到:cond_0

"if-nez vA, :cond_0" 如果vA!=0则跳转到:cond_0

"if-ltz vA, :cond_0" 如果vA<0则跳转到:cond_0

"if-lez vA, :cond_0" 如果vA<=0则跳转到:cond_0

"if-gtz vA, :cond_0" 如果vA>0则跳转到:cond_0

"if-gez vA, :cond_0" 如果vA>=0则跳转到:cond_0

注意
