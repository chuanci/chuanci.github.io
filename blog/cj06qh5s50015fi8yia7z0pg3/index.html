逆向popstar

发表于 2016-12-13 | | 本文总阅读量 次

本文旨在探讨移动开发安全，不以破解获利。x

《Popstar》一款非常容易的消除类游戏，游戏的规则很简单：只需点击两个或两个以上颜色相同的方块即可消除。没有时间限制，每个阶段都有一个目标，完成即可进入下一关。简单轻松的治愈系游戏PopStar，每天工作结束时玩一把，看着小星星飞来飞去，真的有一种带走疲惫的感觉。

过去pc游戏一发布，几天后就可以从3dm，游侠等网站下到破解版，这是单机游戏，应用类软件的通病，代码流程控制放在客户端。

原版apk

在小米应用上,搜索 popstar

找到一款最近更新的，下载。

popstar原版.apk

提取dex

通过工具提取dex，再转为jar，用gui打开。

这是用的是rx+retrofit2，不愧是2016最流行的开源库之一。

再点开代码一看。。。没混淆

混淆

优点

1，混淆可以减少apk包大小
2，混淆可以增加破解apk花费时间

缺点

混淆可能会制造出新bug，增加测试工作量

在时间允许的情况下，尽量混淆代码。

进入游戏

找到购买界面，进入一看，是微信支付，按返回。

这里获得关键词“ 支付失败：支付失败”

这两个支付失败，按照开发逻辑理解

第一个是结果，即通过判断code生成
第二个是原因，即返回的msg

所以，第一个“支付失败：”就在代码中

搜索“支付失败：”

在PopstarA.class里面我们找到了这个字符串

如果没找到，我们可以提取smali文件，将关键词换成unicode进行搜索

可以看出，请求返回进入onError中。

代码注入

那么我们想执行onSuccess中的代码怎么办？

1，将onSuccess中的代码cope到onError中
2，在onError中，删除原有代码，调用onSuccess

按照路径，找到这个内部类，即PopstarA$5.smali

这里用第一种方案，直接cope代码，避免自己写smali语句

最后将onError中“支付成功：）”的提示换成“破解成功”

重新打包

用工具将smali编译到classes.dex中打包，签名。

这几步都没遇到问题，

签名

在开发中，我们可以通过判断签名的md5来进行一些操作，这可以防止二次打包。

这个md5可以截成几段分开保存，可以写到so里。
